</Note>

Final lets you propose payment actions, such as issuing a card, that a user approves on a Final-hosted page.

## Prerequisites

Before you can call the API, you must complete integrator setup in the dashboard. Both steps are passkey-gated and cannot be performed with an API key.

<Steps>
  <Step title="Setup your integrator profile">
    Complete your **integrator profile**: display name, logo, support URL, terms URL, and privacy policy URL.
    This is what users see on the approval page, and it's required before you can create an integrator key.
  </Step>
  <Step title="Create your API key">
    Create an **integrator API key** in the dashboard. You will need this to be able to create intents.
  </Step>
</Steps>

All requests use the base URL `https://api.final.net/v1` and pass the key as a bearer token:

```
Authorization: Bearer final_live_...
```

## Create an intent

Propose a `card.create` intent. The response includes an `approval_url` and a `funding` block.

```bash
curl -X POST https://api.final.net/v1/intents \
  -H "Authorization: Bearer final_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "kind": "card.create",
    "payload": { "acceptable_funding_assets": ["<asset_id>"] }
  }'
```

<Note>
Assets are CAIP-19 identifiers. Call `GET /v1/inflow_assets` for the funding sources you can list in `acceptable_funding_assets`.
</Note>

## Send the user to approve

Redirect the user to the `approval_url` from the response. They sign in, review your integrator information, and approve. On approval the intent moves toward `funded` and, once funding settles, the card is provisioned.

## Track progress

Watch an intent through its lifecycle in one of two ways. Use whichever fits your backend:

<CardGroup cols={1}>
  <Card title="Server-Sent Events" icon="bolt">
    Open a stream for a single intent and receive events as they happen.

    ```
    GET /v1/intents/{intent_id}/events
    ```
  </Card>
  <Card title="Webhooks" icon="webhook">
    Register an endpoint in the dashboard and subscribe to `intent.*` to receive events across all intents.
  </Card>
</CardGroup>

## Show the card

Once the card exists, mint a short-lived embed URL and render the card details in an iframe on one of your registered origins.

```bash
curl -X POST https://api.final.net/v1/card_refs/{ref}/embed_url \
  -H "Authorization: Bearer final_live_..."
```
